# 🛡️ Дискреционная модель политики безопасности (DAC)

Данный проект представляет собой программную реализацию **дискреционной (избирательной) модели управления доступом** в соответствии с лабораторной работой по дисциплине «Защита информации». Модель основана на **матрице доступа**, где каждому пользователю назначаются права на объекты системы, а также реализована возможность передачи прав.

## ⚙️ Функциональные возможности

- Инициализация системы с заданным количеством пользователей и объектов.
- Один из пользователей назначается **администратором** (обладает всеми правами на все объекты).
- Для остальных пользователей права генерируются **случайным образом** (набор из прав: чтение, запись, передача прав, либо полные права, либо запрет).
- Поддержка операций:
    - `read` – чтение объекта (проверка права);
    - `write` – запись в объект;
    - `grant` – передача своего права другому пользователю (при наличии права `GRANT`);
    - `show` – отображение текущей матрицы доступа;
    - `quit` – выход из системы.
- При передаче прав выполняется проверка возможности создания нового администратора (если пользователь получает все права на все объекты, операция блокируется).
- После каждой операции выводится сообщение об успехе или отказе.
- Матрица доступа модифицируется только при успешной операции `grant`.

## 🧰 Технологии

- **Java 17** (или выше)
- **Gradle** (многомодульная структура проекта)
- Документирование кода – **Javadoc**

### Пример диалога

```
Жду ваших указаний > read
Над каким объектом производится операция? Объект_1
Операция прошла успешно.

Жду ваших указаний > write
Над каким объектом производится операция? Объект_2
Отказ в выполнении операции. У Вас нет прав для ее осуществления.

Жду ваших указаний > grant
Право на какой объект передается? Объект_3
Какое право передается? Чтение
Какому пользователю передается право? Petr
Операция прошла успешно.

Жду ваших указаний > show
Матрица доступа:
Ivan (админ): Объект_1=[Чтение, Запись, Передача прав] Объект_2=[...] ...
...
Жду ваших указаний > quit
Работа пользователя Ivan завершена. До свидания.
```

## 🗂️ Структура проекта

Проект организован как многомодульный Gradle-проект (каждая лабораторная может быть отдельным модулем). Основной модуль содержит:

```
src/main/java/ru/rsreu/sovynhik/pract_03/
├── config          – константы (Constants.java)
├── model           – классы сущностей (User, SystemObject, Right, AccessEntry)
├── repository      – репозитории (UserRepository, ObjectRepository, AccessMatrixRepository)
├── service         – сервисы (AuthenticationService, AuthorizationService, AccessMatrixService)
├── ui              – консольный интерфейс (ConsoleUI)
└── SecuritySystemApplication.java – точка входа
```
